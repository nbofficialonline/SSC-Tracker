# Careerwill SSC CGL Tracker

A full-stack SSC CGL topic tracker tailored for the Careerwill batch. Students track topic completion, study time, and streaks, while admins manage users, access, and site settings.

## Features
- Student dashboard with subject-wise progress and priority tracking
- Topic checklists with filters, bulk mark, and CSV export
- Study timer with session history and streak calculation
- Admin console for users, roles, expiry, logs, and settings
- Secure session auth with CSRF protection and rate limiting
- Single-page UI in public/index.html (no build step)

## Tech Stack
- Node.js + Express
- MongoDB + Mongoose
- Sessions with connect-mongo
- Frontend: Vanilla JS + CSS (single-page app)

## Quick Start
1. Install dependencies:
   - npm install
2. Create a .env file in the project root (see below).
3. Run the server:
   - npm run dev
4. Open http://localhost:3000

## Environment Variables
Create a .env file with:
- MONGODB_URI=your_mongodb_connection_string
- SESSION_SECRET=your_session_secret
- CSRF_SECRET=your_csrf_secret
- SITE_URL=http://localhost:3000
- NODE_ENV=development
- PORT=3000

For initial admin seeding:
- ADMIN_USERNAME=admin
- ADMIN_PASSWORD=your_admin_password

## Scripts
- npm run dev: Start dev server with nodemon
- npm start: Start production server
- npm run seed-admin: Seed the first admin account
- node scripts/build-topics.js: Build data/topics.json from data/raw

## Data and Topics
- data/raw contains the source syllabus files
- data/topics.json is generated by scripts/build-topics.js
- Admins can rebuild topics from the UI (Admin > Settings > Rebuild)

## Admin Workflow
1. Seed admin (first time only):
   - npm run seed-admin
2. Login via the Admin tab on the login page
3. Manage users, expiry, and progress from the Admin console

## Performance Notes
- Gzip compression enabled in Express
- Local UI caching for faster subject reloads
- Content-visibility enabled for heavy lists
